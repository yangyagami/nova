## 1. 产品定位

### 1.1 一句话定义
**Nova 是一款面向个人创作者的桌面端 AI 恐怖小说生成与润色工具，支持从一句话创意到完整长篇小说 / 短篇怪谈集的全自动产出，并提供精细的人工润色能力。**

### 1.2 目标用户
- **核心用户**：希望低成本、高产能在网络文学平台（番茄小说、知乎盐选、短剧爽文平台、公众号）上架恐怖题材作品的个人创作者。
- **次要用户**：恐怖题材爱好者，用于个人创作、同人写作。

### 1.3 用户痛点
| 痛点 | Nova 的解法 |
|---|---|
| AI 写长篇人物/设定前后矛盾 | 自动维护"设定库"，每章生成时按需注入 |
| 一眼 AI 味，平台审核被拒 | 内置"去 AI 味"二次润色管线 |
| 恐怖题材容易写成血浆爽文，没氛围 | 恐怖专用 Prompt 模板，强制铺垫节奏 |
| 网页版工具数据在云端，怕泄漏/丢稿 | 桌面应用，本地 SQLite 存储 |
| 通用 AI 工具上下文短，长篇撑不住 | 章节摘要压缩 + 滑动窗口上下文 |
| **想写短篇合集但工具只支持长篇小说** | **支持「短篇怪谈集」模式，每篇独立生成，无固定主角** |

### 1.4 商业目标（用户侧）
帮助单个用户每月稳定产出 **2-4 本 10-30 万字** 的恐怖中长篇，或 **4-8 本 3-8 万字** 的短篇怪谈集，月稿费/分成目标 5000-30000 元。

## 3. 用户故事（User Stories）

### US-01：新手创作者首次使用
> 作为一个第一次用 Nova 的用户，我希望能在 5 分钟内完成 API 配置、新建项目并看到第一篇内容生成出来，以便快速验证产品价值。

**验收标准：**
- 首次启动有引导页，3 步完成 API Key 配置
- 引导用户基于"民俗恐怖"模板新建示例项目，自动填充示例创意
- 可选择「长篇单行」或「短篇怪谈集」两种模板
- 点击"生成"，30 秒内开始流式输出

### US-02：长篇全自动产出
> 作为追求产能的用户，我希望配置好大纲后，让 Nova 在后台自动生成全部章节，我离开电脑后回来就能看到成稿。

**验收标准：**
- "批量生成"按钮启动后，按章节顺序生成全部章节
- 每章生成后自动更新设定库与摘要
- 中途失败自动重试，连续 3 次失败暂停并通知用户
- 生成完成后桌面通知 + 总用量统计

### US-03：短篇怪谈集一键生成
> 作为一个想写「都市怪谈集」的作者，我希望输入合集主题和故事数量后，Nova 自动生成多篇独立短篇故事，每篇有自己的主角、设定和完整剧情。

**验收标准：**
- 新建项目时选择「短篇怪谈集」模式
- 输入合集主题（如"深夜都市的 10 个诡异传闻"）
- 设定故事数量和每篇字数
- 点击"生成大纲" → 出现 N 张故事卡片，每张含独立标题、创意、场景、主角
- 可逐篇查看/编辑每篇的细纲
- 点击"生成全部故事" → 串行生成，每篇为完整短篇（有头有尾）
- 生成完成后，每篇故事独立可编辑和导出

### US-04：精修单章/单篇
> 作为对质量有要求的用户，我希望能选中某段文字，让 AI 增强恐怖氛围，并保留修改前的版本以便对比。

**验收标准：**
- 编辑器选中文字 → 右键菜单出现 5 个 AI 操作
- AI 改写以"对比视图"显示原文/新文，可接受/拒绝/再生成
- 接受后原版本进入"版本历史"

### US-05：避免人物崩坏（长篇模式）
> 作为长篇作者，我希望主角的性格和外貌在第 50 章和第 1 章保持一致。

**验收标准：**
- 设定库中可"锁定"人物卡关键字段
- 每章生成 prompt 自动注入相关人物卡
- 编辑器侧边栏可随时查看/搜索设定库

### US-06：成本可控
> 作为预算敏感的用户，我希望知道每次操作大概花多少钱。

**验收标准：**
- 设置页显示累计 token 消耗与人民币估算
- 单章/单篇生成后弹窗显示本次消耗
- 批量生成前预估总成本，用户确认后开始

## 4. 关键非功能需求

| 维度 | 要求 |
|---|---|
| **性能** | 单章 3000 字生成 ≤ 60 秒；单篇 5000 字短篇生成 ≤ 90 秒；编辑器加载 30 万字小说 ≤ 2 秒 |
| **可靠性** | 生成中崩溃后重启可恢复未保存内容；SQLite 每 30 秒自动保存 |
| **安全** | API Key 用系统 keychain 存储（macOS Keychain / Windows Credential Manager） |
| **隐私** | 100% 本地存储，除调用 DeepSeek API 外无任何外部网络请求 |
| **包体积** | 安装包 ≤ 30MB |
| **平台** | Windows 10+、macOS 12+、Linux（Ubuntu 20.04+） |
| **离线能力** | 除 AI 生成外所有功能（编辑、查看、导出）可离线 |

## 5. 商业可行性约束

### 5.1 平台合规风险
- **番茄小说 / 七猫**：当前对 AI 容忍度较高，但要求"过审"（无敏感词、无露骨描写）。Nova 内置敏感词检查。
- **起点 / 晋江**：明令禁止纯 AI 创作，**Nova 不推荐用于此类平台**。
- **公众号 / 知乎盐选**：以质量为导向，需重度人工润色，Nova 的"润色编辑器"是核心。短篇怪谈集非常适合知乎盐选。
- **短剧/小程序爽文平台**（河马、九州等）：最适合，套路化强。

### 5.2 成本测算（DeepSeek 现价）
- DeepSeek-Chat：输入 ¥0.5/M tokens，输出 ¥2/M tokens（缓存命中更便宜）
- **长篇模式**：一本 20 万字小说 ≈ 40 万 token 输出 + 200 万 token 输入 → **单本成本约 ¥2-5**
- **选集模式**：一本 5 万字（10 篇 × 5000 字）短篇集 ≈ 10 万 token 输出 + 50 万 token 输入 → **单本成本约 ¥1-2**
- 月产 4 本长篇 + 4 本短篇集成本 < ¥30

- ✅ 用户能在 1 小时内独立产出第一本 5 万字短篇 / 3 万字怪谈集
- ✅ 生成稿件经人工润色 2 小时内可投稿
